### 用例3 电器购物

| ID             | UC3                                                          |
| -------------- | ------------------------------------------------------------ |
| 名称           | 电器购物                                                     |
| 创建者         | 杨郁芩                                                       |
| 最后一次更新者 | 杨郁芩                                                       |
| 创建日期       | 2018.10.23                                                   |
| 最后更新日期   | 2018.10.23                                                   |
| 参与者         | 遥控终端使用者                                               |
| 触发条件       | 遥控终端使用者进入电器商城                                   |
| 前置条件       | 遥控终端使用者已经登录，并且绑定账号                         |
| 后置条件       | 产生购物订单                                                 |
| 优先级         | 中                                                           |
| 正常流程       | 1. 遥控终端使用者点击某一电器进入浏览内容<br />2. 系统展示该电器的规格和具体信息<br />3. 遥控终端使用者根据关键字对电器进行搜索<br />4. 系统显示搜索结果<br />5. 遥控终端使用者将该电器加入购物车<br />6. 系统提示加入购物车成功<br />7.遥控终端使用者在购物车中选择需要付款的商品并提交订单<br />8. 系统选择需要付款的商品<br />9. 遥控终端使用者选择优惠策略<br />10. 系统显示已经选择的优惠策略<br />11. 遥控终端使用者填写收货信息<br />12. 系统显示用户填写的收货信息<br />13. 遥控终端使用者选择付款方式并进行付款<br />14. 系统提示付款成功并生成订单 |
| 扩展流程       | 3a. 遥控终端使用者输入关键字无搜索结果<br />&emsp;1.   系统提示无结果<br />5a. 遥控终端使用者未登录<br />&emsp;1.   系统提示用户登录<br />&emsp;2.   返回登录界面<br />5b. 该电器无库存<br />&emsp;1.    系统提示该电器已经无库存，并通知后台管理员下架商品<br />9a. 遥控终端使用者选择的优惠不符合规则<br />&emsp;1.   系统提示不能使用该优惠<br />11a. 遥控终端使用者付款失败<br />&emsp;1.    系统提示用户更换付款方式<br />&emsp;2.    返回付款界面<br />11b. 遥控终端使用者提交订单后却未付款<br />&emsp;1.    系统保存该未付款订单，并提示遥控终端使用者24小时内未付款将自动取消订单 |
| 特殊需求       | 需要引入第三方支付功能<br />搜索框中应有自动补全功能         |

| 创建日期       | 2018.10.23                                                   |

| 优先级         | 高                                                           |
| 正常流程       | 1、用户在购物界面选择我的购物清单<br />2、系统展示用户的商品列表<br />3、用户选中商品并选择客服<br />4、系统进入用户-客服消息界面<br />5、用户发送消息<br />6、系统将消息传输给后台管理员并提示出现客服消息<br />7、后台管理员选择客服消息<br />8、系统显示所有消息列表<br />9、后台管理员双击消息<br />10、系统打开用户-客户消息界面<br />11、后天管理员回复消息<br />12、系统将消息传输到遥控终端用户并提示出现客服消息<br />13、用户选择消息列表<br />14、系统显示所有客服消息<br />15、用户选中某客服<br />16、系统显示用户-客户消息界面，展示消息来往<br />循环5~16步直到用户与客服完成货沟通 |
| 扩展流程       | 1a、用户选择消息<br />          1、系统显示客服消息列表<br />          2、用户选择某客服<br />          跳到正常流程步骤5继续执行<br /> |
| 特殊需求       | 暂无                                                         |
